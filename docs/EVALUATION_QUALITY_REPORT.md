## 标注边界

`annotations.jsonl` 是最终评估集的唯一 canonical GT 文件。标注边界如下：

- 空标注不进入评估。
- 手写拍照、多行区域、页面图像、脚注、数字和彝汉混排不因难度高而排除。
- 多行样本保留换行，避免把区域图或整页图强行压成单行。
- 拉丁注音只在人工标注可见时保留；人工标注无拉丁注音的样本用于检查模型是否出现多余拉丁字母输出。
- 评估样本不进入训练包，训练侧只使用排除清单规避精确重合。

## 数据规模与粒度

最终数据集覆盖真实评估所需的主要输入粒度，但项目仍把单行图、区域图、整页图和整页切割对照分口径解读，避免把行图识别、区域换行和整页版式能力混为一个指标。

| 样本类型 | 样本数 |
|---|---:|
| 单行图 | 470 |
| 区域图 | 119 |
| 整页图 | 169 |

行图用于观察文字识别稳定性；区域图和整页图用于暴露漏行、版面边界、阅读顺序和非标注换行问题。页面切割对照单独报告切割前后指标，不与 line / region OCR 平均数混合。

## 样本纳入标准

`758` 条样本全部计入最新评估集口径。纳入标准如下：

| 标准 | 口径 |
|---|---|
| 可见文本 | 图片中有可判读的规范彝文、汉字、数字、标点或混排文本。 |
| 图片有效 | 图片路径存在，且与 `annotations.jsonl` 一一对应。 |
| 标注有效 | GT 非空，不含占位符和 replacement char。 |
| 唯一性 | 样本 ID 不重复，图片引用不重复。 |
| 真实来源 | 评估样本来自真实资料，不使用合成样本作为评估材料。 |

## 评估核验入口

公开仓库保留以下核验材料：

```text
NuosuBburma_OCR_Evaluation_Set/README.md
docs/EVALUATION_DATASET.md
evaluation/summary.md
evaluation/summary.json
evaluation/tables/
evaluation/charts/
evaluation/raw/submission_model_result.jsonl
scripts/eval_nuosubburma.py
scripts/analyze_submission_eval.py
```

Hugging Face 评估集保留：

```text
annotations.jsonl
images/
README.md
samples.csv
dataset_summary.json
```

评估脚本会输出整体归一化编辑距离、空白处理后的编辑距离、Unicode 规范化后的编辑距离，以及只看彝文、只看汉字、只看数字的编辑距离。

脚本还会统计替换符、公式化片段、多余拉丁字母和异常长输出等输出风险指标。

首页不把完全匹配率作为主指标，因为它对空白、换行和标点过于敏感。

## 当前结果摘要

历史提交模型已使用同一评估脚本统计 `603` 条 OCR 主指标。最终 `758` 条整合评估集作为原始模型与 LoRA 模型的统一评估口径。

| 指标 | 当前 LoRA 模型 |
|---|---:|
| 历史 `603` 条样本数 | 603 |
| 历史 `603` 条平均归一化编辑距离（Avg NED，越低越好） | 0.036068 |
| 历史 `603` 条忽略空白后的平均编辑距离 | 0.034219 |
| 历史 `603` 条 NFKC+WS Avg NED | 0.033964 |
| 历史 `603` 条 Yi-only Avg NED | 0.038309 |
| 历史 `603` 条 Han-only Avg NED | 0.022447 |
| 历史 `603` 条 Digit-only Avg NED | 0.139918 |
| 历史 `603` 条 replacement / LaTeX / extra Latin / long_pred | 0 / 2 / 0 / 0 |

结果可由 `scripts/run_eval.sh` 和 `scripts/analyze_submission_eval.py` 复跑。逐条输出保存在 `evaluation/raw/submission_model_result.jsonl`，用于解释分支选择和错误风险。
