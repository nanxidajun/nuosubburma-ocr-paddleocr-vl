## 标注流程

评估集采用“模型预标注 + 人工核对”的方式构建。预标注用于降低人工录入成本，最终标准答案仍以人工核对为准。

```text
真实来源图片整理
-> 页面切割或区域裁剪
-> 中间模型生成预标注草稿
-> 人工逐条核对标准答案
-> 删除坏图、单字碎片或不可稳定判读样本
-> 统一元数据和 annotations.jsonl
-> 生成分布统计与复查页面
```

标注规则：

- 空标注和占位符标注不进入评估。
- 手写拍照、多行区域、页面图像、脚注、数字和彝汉混排不因难度高而简单删除。
- 多行样本保留换行，避免把区域图或整页图强行压成单行。
- 拉丁注音只在人工标注可见时保留；人工标注无拉丁注音的样本用于检查模型是否出现多余拉丁字母输出。
- 评估样本不进入训练包，训练侧只使用排除清单规避精确重合。

## 数据规模与粒度

最终数据集覆盖真实评估所需的主要输入粒度，但项目仍把单行图、区域图、整页图和整页切割对照分口径解读，避免把行图识别、区域换行和整页版式能力混为一个指标。

| 样本类型 | 样本数 |
|---|---:|
| 单行图 | 470 |
| 区域图 | 119 |
| 整页图 | 169 |

行图用于观察文字识别稳定性；区域图和整页图用于暴露漏行、版面边界、阅读顺序和非标注换行问题。页面切割对照单独报告切割前后指标，不与 line / region OCR 平均数混合。

## 难度合理性

难度分布如下：

| 难度 | 样本数 |
|---|---:|
| 简单 | 83 |
| 中等 | 467 |
| 困难 | 208 |

难度不是按模型预测结果倒推，而是结合来源、视觉质量、输入粒度和文本混合情况标注。

最新评估集保留了手写拍照、区域图、整页图、页面图像、旧印刷、数字、脚注和彝汉混排样本，因此不会只评估清晰 PDF 行图。

## 删除与归档记录

构建过程中有重复封面行图、旧截屏页面和人工点名删除图片被删除、替换或归档，原因包括：

- 行切分版本与完整原图重复，保留完整原图或更稳定版本。
- 单字切片信息量太低，不作为评估样本。
- 用户补充的更清晰副本替换旧版本。
- 页面图像使用人工整理后的完整标准答案。

这些删除用于去重、替换坏图和移除低信息量样本。保留的 `758` 条样本全部计入最新评估集口径。


Hugging Face 评估集保留：

```text
annotations.jsonl
images/
README.md
samples.csv
dataset_summary.json
```

评估脚本会输出整体归一化编辑距离、空白处理后的编辑距离、Unicode 规范化后的编辑距离，以及只看彝文、只看汉字、只看数字的编辑距离。

脚本还会统计替换符、公式化片段、多余拉丁字母和异常长输出等输出风险指标。

首页不把完全匹配率作为主指标，因为它对空白、换行和标点过于敏感。
