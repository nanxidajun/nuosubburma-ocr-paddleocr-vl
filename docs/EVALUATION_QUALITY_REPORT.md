## 标注流程

评估集采用“模型预标注 + 人工核对”的方式构建。预标注用于降低人工录入成本，最终标准答案仍以人工核对为准。

```text
真实来源图片整理
-> 页面切割或区域裁剪
-> 中间模型生成预标注草稿
-> 人工逐条核对标准答案
-> 删除坏图、单字碎片或不可稳定判读样本
-> 统一元数据和 annotations.jsonl
-> 生成分布统计与复查页面
```

标注规则：

- 空标注和占位符标注不进入主评估。
- 手写拍照、多行区域、页面图像、脚注、数字和彝汉混排不因难度高而简单删除。
- 多行样本保留换行，避免把区域图或整页图强行压成单行。
- 拉丁注音只在人工标注可见时保留；人工标注无拉丁注音的样本用于检查模型是否出现多余拉丁字母输出。
- 评估样本不进入训练包，训练侧只使用排除清单规避精确重合。

## 数据规模与粒度

最终数据集覆盖真实评估所需的主要输入粒度，但项目仍把单行图、区域图和整页图分开解读，避免把行图识别、区域换行和整页能力混为一个指标。

| 样本类型 | 样本数 |
|---|---:|
| 单行图 | 515 |
| 区域图 | 84 |
| 整页图 | 4 |

行图用于观察文字识别稳定性；区域图和整页图用于暴露漏行、版面边界、阅读顺序和非标注换行问题。复杂整页压力测试与主评估集分开维护。

## 多样性

评估集从场景、文字混合和数字三个维度记录分布。

| 文字混合 | 样本数 |
|---|---:|
| 彝文 | 321 |
| 彝汉混排 | 265 |
| 带拉丁字母的彝汉混排 | 17 |

| 是否含数字 | 样本数 |
|---|---:|
| 含数字 | 81 |
| 不含数字 | 522 |

数字、页码和编号在结果中单独报告，因为这类字符在文字识别输出里容易受格式、换行和标点影响。

## 难度合理性

难度分布如下：

| 难度 | 样本数 |
|---|---:|
| 简单 | 172 |
| 中等 | 324 |
| 困难 | 107 |

难度不是按模型预测结果倒推，而是结合来源、视觉质量、输入粒度和文本混合情况标注。

主集保留了手写拍照、区域图、页面图像、旧印刷、数字、脚注和彝汉混排样本，因此不会只评估清晰 PDF 行图。

## 删除与归档记录

构建过程中有 `27` 条临时整理样本被删除、替换或归档，原因包括：

- 行切分版本与完整原图重复，保留完整原图或更稳定版本。
- 单字切片信息量太低，不作为主评估样本。
- 用户补充的更清晰副本替换旧版本。
- 页面图像使用人工整理后的完整标准答案。

这些删除用于去重、替换坏图和移除低信息量样本。保留样本全部计入主评分。

## 评估复查入口

公开仓库保留以下复查材料：

```text
NuosuBburma_OCR_Evaluation_Set/README.md

```

Hugging Face 评估集保留：

```text
annotations.jsonl
images/
README.md
samples.csv
dataset_summary.json
```

评估脚本会输出整体归一化编辑距离、空白处理后的编辑距离、Unicode 规范化后的编辑距离，以及只看彝文、只看汉字、只看数字的编辑距离。

脚本还会统计替换符、公式化片段、多余拉丁字母和异常长输出等输出风险指标。

首页不把完全匹配率作为主指标，因为它对空白、换行和标点过于敏感。
