# 评估集标注与质量检查

本报告说明 1,030 张真实评估图片的标注原则、分级人工质检流程和发布前一致性审计。目标是让评估结果建立在可核验、无污染的标准答案之上。

## 标注原则

标准答案（GT）只记录图中**可见文字**，按视觉阅读顺序转写规范彝文、汉字、拉丁字母、数字和标点，保留必要的行结构和原有字符，**不翻译、不改写、不补全**图中不存在的内容。图像退化或噪声不改变 GT。每条样本只有一份正式 assistant GT，位于数据集 `annotations.jsonl` 中。

## 分级人工质检流程

正式 GT 采用分级人工质检体系，模型输出只作为预标注和差异提示，不能自动覆盖人工结论：

1. **预标注**——先生成预标注，但不把预标注直接作为最终标准答案。
2. **多版本交叉检查**——用多轮训练得到的不同模型版本产生识别结果，辅助发现潜在标注错误。
3. **字符级对齐高亮**——通过字符级对齐工具高亮替换、漏字和多字位置，集中检查模型输出与当前标注的差异。
4. **首轮人工核查**——项目负责人对照原始图片检查高亮出的差异并修正错误。
5. **母语者复核**——将疑难、不确定或容易混淆的规范彝文样本交由熟悉规范彝文的母语者核查。
6. **人工裁决为准**——最终标准答案由人工确认，以原图和人工判断为准。

字符级对齐工具支持逐条标记“预测错”“GT 错”“切图问题”“不确定”等状态并填写人工正确答案，复核结论以增量方式应用到评估包，不覆盖原始记录。[在线可视化](https://huggingface.co/spaces/nanxidajun/NuosuBburma-OCR-Evaluation-Set) 提供图片与标准答案的逐条对照，便于公开复查。

## 发布前一致性审计

在人工质检之外，发布前对全部 1,030 条样本执行自动一致性审计：

| 检查项 | 结果 |
|---|---:|
| 正式样本 / 正式图片 | 1,030 / 1,030 |
| 唯一 ID / 唯一图片引用 | 1,030 / 1,030 |
| 缺失图片 / 孤立图片 | 0 / 0 |
| 空 GT / 占位符 GT / 待定样本 | 0 / 0 / 0 |
| 重复 ID / 重复图片哈希 | 0 / 0 |
| 图片与标注映射异常 | 0 |
| 图片尺寸与元数据不一致 | 0 |
| 合成评估图片 | 0 |
| 审计状态 | `AUDIT_PASSED` |

审计摘要与标注哈希记录在数据集 `dataset_summary.json` 中，可对照复核。

## 训练隔离

- 评估图片、GT、预测、错误类型和统计**不进入**训练或开发数据。
- 训练生成器不读取评估目录、书名、样本 ID 或图像裁切。
- 形近字集合、字体、上下文、退化条件和训练权重不依据评估错例选择。
- 评估集只在模型训练和导出完成后用于报告结果。

## 报告边界

本报告只陈述已执行的质检流程与已通过的发布审计。母语者复核针对疑难、不确定或易混的规范彝文样本，非全量；报告不含尚未统计的标注一致率、纠错条数或复核轮数等指标，也不声称双盲独立标注。
