# 训练数据构建报告

本文档只说明训练包如何构建、标注和质检；隔离检查单列说明。三阶段 LoRA 调优策略见 [模型与训练说明](MODEL_AND_TRAINING.md)，评估集真实性见 [评估集说明](EVALUATION_DATASET.md)。

训练包、开发诊断集和最终评估集分开使用。训练包用于微调模型；开发诊断集用于比较训练分支；最终评估集用于报告模型效果。本文档中的“人工整理”只指训练侧标签整理，不等同于评估集标准答案。

## 评分项对应

| 官方子项 | 本项目对应材料 |
|---|---|
| 采集流程规范性 | 训练材料按真实训练材料、训练侧合成样本、文本不变的视觉变化样本和训练侧约束样本分组；训练包清单保留构建脚本、包名、随机种子和清理记录 |
| 标注规范完整性 | 真实训练材料使用训练侧整理文本；合成样本使用规则生成标签；视觉变化样本不改变文本内容 |
| 质量控制机制 | 检查缺图、空标签、替换符、公式化片段标签、反斜杠和高风险样本比例 |
| 数据统计分析 | 训练包清单记录训练行数、真实/合成构成、视觉变化样本比例、风险样本上限和主要类别分布 |
| 隔离检查 | 评估图片路径、文件名和样本编号不进入训练包；评估集标准答案只用于检查完全重合条目 |

## 构建原则

规范彝文识别是低资源文字场景。真实训练样本有限，难以覆盖 1165 个规范彝文字符、字体变化、旧印刷状态、彝汉混排、数字和页面边界。合成样本可以补覆盖，但如果不设上限，模型又容易学出多余拉丁字母、公式化片段、替换符和异常长输出。

因此训练包采用四类材料：

```text
真实训练材料
+ 训练侧合成样本
+ 文本不变的视觉变化样本
+ 训练侧约束样本
```

核心规则：

- 真实训练材料用于确定任务边界。
- 训练侧合成样本只用于补字符和格式覆盖，不进入最终评估集。
- 视觉变化样本只改变字体、清晰度和旧印刷状态，不改变文本标签。
- 拉丁注音、脚注、多行区域等高风险样本设置上限。
- 评估集图片、样本编号和标准答案不进入训练包。

三阶段调优中的第一阶段主要使用真实训练材料验证可行性；第二阶段用训练侧合成样本补字符覆盖，并用视觉变化样本补字体、清晰度和旧印刷状态；第三阶段只用固定开发诊断集比较分支输出，不改变训练包。

## 训练数据来源

训练数据来自两类入口：真实训练材料整理和训练侧规则生成。

| 来源类型 | 内容 | 在训练中的作用 |
|---|---|---|
| 真实旧书/扫描件裁切行 | 旧书、资料选译、语法书、工具书等训练材料 | 提供真实字体、旧印刷、长行、彝汉混排和版式边界 |
| 真实短行与混排行 | 纯彝文、彝汉混排、标题、目录、短句 | 学习基本 OCR 输出格式和常见行图形态 |
| 手写拍照与页面照片 | 训练侧采集或整理的手写拍照、页面照片、实拍照片 | 提供真实用户输入路径的初步泛化信号 |
| 合成低频字覆盖 | 按字符频率和规范彝文字符集生成 | 补真实训练中缺失或极低频的字形 |
| 字符覆盖样本 | 面向低频字符、声调变体和字体变化 | 降低字符覆盖不足带来的不稳定 |
| 旧印刷与页面退化 | 低对比、轻微模糊、压缩、旧印刷缺损等 | 扩大视觉域，而不是只适应清晰行图 |
| 多行区域与脚注约束样本 | 段落、换行、脚注和注音邻近样本 | 训练换行和完整性，同时观察公式化片段和长输出风险 |

公开仓库不直接发布训练图片包。仓库保留提交模型对应的训练配置和训练包清单，用于核验训练包构成：

```text
configs/paddleocr-vl_lora_16k_nuosubburma_v5_16.yaml
configs/paddleocr-vl_lora_export_nuosubburma_v5_16.yaml
configs/train_data_manifest_v5_16.json
```

## 标签来源

训练标签只来自训练侧材料或训练侧规则，不使用评估集标准答案。

| 样本类型 | 标签来源 | 进入训练前检查 |
|---|---|---|
| 真实训练材料 | 训练材料对应的人工整理文本 | 图片可判读、路径存在、标签非空、无替换符 |
| 训练侧合成样本 | 渲染文本本身就是标签 | 字符集、标点、长度和输出格式检查 |
| 视觉变化样本 | 继承原训练标签，只改变字体、清晰度和旧印刷外观 | 标签不变，检查渲染图片、空标签、反斜杠和公式化片段标签 |
| 训练侧约束样本 | 训练侧规则构造的边界文本 | 按通道限额并入，避免某类格式过量 |

真实训练材料流程：

```text
训练材料整理
-> 切成行图或区域图
-> 整理训练侧文本标签
-> 删除坏图、不可判读样本或重复样本
-> 检查路径、空标签和格式
-> 写入训练包
```

合成与视觉变化样本流程：

```text
训练侧字符/版式规则
-> 生成文本标签
-> 渲染为图片
-> 同一文本生成不同字体、清晰度和旧印刷外观
-> 检查图片、标签和高风险格式
-> 按通道上限并入训练包
```

## 高风险通道控制

高风险样本控制不是评估结果，而是训练包构建时的比例控制。目的在于避免某类格式被模型学成默认输出习惯。

| 通道 | 当前处理 |
|---|---|
| 拉丁注音 | 不放大拉丁注音视觉变化样本；保留必要的字典/注音样本，但避免把拉丁尾巴扩成默认输出 |
| 脚注/符号 | 只保留少量字面符号约束样本，避免增加公式化片段先验 |
| 多行区域 | 保留多行完整性训练，但设置上限，避免模型把单行也输出成多行 |
| 字符覆盖 | 使用训练侧覆盖规则补低频字符和字体变化 |
| 旧印刷退化 | 用文本不变的视觉变化样本补旧书图像状态，不改变文字标签含义 |

## 隔离检查

本节只说明训练包和评估集是否混用，不用评估集证明训练包质量。

| 检查项 | 结果 |
|---|---:|
| 评估图片路径命中训练包 | 0 |
| 评估图片文件名命中训练包 | 0 |
| 评估样本编号命中训练包 | 0 |
| 删除的完全重合标签行 | 2 |
| 保留的完全重合标签行 | 7 |

2026-06-22 的清理记录显示，已删除 `2` 条来自手写训练子集、且与评估集完全重合的标签行。清理后仍有 `7` 条完全重合标签，来源为继承的旧训练行，不来自当前新增视觉变化样本或手写训练子集。

训练侧遵守以下规则：

- 不按评估集错例反向增补训练样本。
- 不把评估集标准答案拆成字符清单加入训练。
- 不使用待定占位样本。
- 评估集标准答案只用于排除完全重合条目。
- 新样本来自原始合成标签、训练侧频率规则、字符集规则或版式规则。
