# 合成训练数据构建报告

本文档说明最终模型两个训练阶段的合成数据来源、生成规则、配额、标签和隔离检查。训练集用于模型微调；1,030 张真实评估图片及其 GT 只用于最终评估，不进入训练或开发数据。

## 数据边界

两个阶段的正式训练数据均为独立合成数据。允许的输入只有规范字表、字体文件、生成器代码和预先冻结的通用形近字表。

| 可使用 | 禁止使用 |
|---|---|
| 规范彝文 Unicode 字表 | 评估图片及其裁切图 |
| 常用汉字表 | 评估 GT |
| 锁定字体文件 | 模型在评估集上的预测 |
| 训练侧文本与版式规则 | 评估错误、错误统计和页级结果 |
| 独立生成的通用形近字表 | 根据评估错例反向选择字符、字体或退化条件 |

训练构建不读取评估书名、样本 ID 或来源目录。开发数据同样由独立 seed 合成，并在训练前冻结。

## 第一阶段数据

第一阶段用于建立规范彝文、汉字、拉丁文字、数字和标点的混合 OCR 能力。

### 正式规模

| 数据家族 | Train | Dev |
|---|---:|---:|
| 通用核心 | 18,000 | 800 |
| Latin 补充 | 400 | 80 |
| 残墨与真字退化成对样本 | 400 | 80 |
| 合计 | 18,800 | 960 |

### 通用核心

通用核心由以下独立规则组合生成：

- 规范彝文字符来自锁定的 Unicode 字表，覆盖标准字符与声调变体。
- 汉字来自锁定的常用汉字表。
- 使用四款彝文字体，覆盖黑体、宋体和手写风格。
- 页面由单行、区域和多行版式规则生成，GT 按视觉阅读顺序写入。
- 带圈编号 `①-⑳` 按字体和行首/行中/行尾位置注入核心，不额外增加样本数。
- 2,162 个通用形近字对作为字符覆盖的一部分进入核心，方向按固定训练侧规则排程。

生成器先创建文本和版面，再渲染清晰图，最后按训练侧 profile 施加图像状态变化。图像与 GT 由同一条生成记录绑定。

### Latin 补充

400 条训练样本由四种内容构成：

| 内容 | Train |
|---|---:|
| 通用词形 | 260 |
| 姓名 | 60 |
| 年份与数字 | 40 |
| 缩写与代码 | 40 |

环境分为纯 Latin 80、彝文 + Latin 160、彝汉 + Latin 160，覆盖 A-Z/a-z。开发集按相同比例缩放为 80 条。

### 残墨与真字退化

训练集包含 200 组成对样本，共 400 张图：

- 100 组外部墨点对照：墨点不与正文笔画重合。
- 100 组真字退化对照：使用轻度模糊与对比度变化模拟真实字形退化。

开发集包含 40 组、80 张图。每组清晰图与退化图的 GT 逐字节相同，用于让模型区分图像噪声、残墨和真实字符。

## 图像状态策略

合成数据不是无限叠加随机噪声，而是使用可审计的 profile：

| 规则 | 设置 |
|---|---|
| 清晰/旧化覆盖 | 同时保留清晰、轻旧化和重但可读条件 |
| 重退化占比 | 30% |
| 轻/重核心操作预算 | 最多 1 / 2 个 |
| shear / gamma | 禁用 |
| 字体 | 四款锁定彝文字体 |
| 阅读顺序 | 使用版面坐标确定 |

人工抽查曾确认七类退化路径过重。最终策略保留这些类别，但把输出与同一张清晰图进行固定比例插值：其中一类强度为 0.25，其余六类为 0.80。这样保留退化类型和曝光次数，同时避免正文接近不可读。

每张非清晰样本记录 profile、操作顺序、字体、版式和关键退化参数。相同 seed 的重复构建必须产生一致的清单和图片哈希。

## 第二阶段形近字数据

第二阶段在保留第一阶段数据的基础上，新增 4,324 张形近字 A/B 页面。

### 形近字集合

形近字集合由规范彝文字形在多字体、多尺度视图下独立计算得到，不读取评估错误。最终使用 2,162 个通用形近字对，来源包括多字体字形聚类、声调弧线关系、字体特定聚类和覆盖补足。

### A/B 页面规则

每个形近字对生成两张匹配页面：

~~~text
A 页面：目标位置使用字符 A
B 页面：同一目标位置使用字符 B
~~~

两张图保持以下条件完全相同：

- 背景与版式；
- 上下文文字；
- 目标字符位置；
- 字体；
- 清晰或轻旧化条件；
- 非目标字符和非目标像素区域。

只有目标字符邻域允许变化。目标字符左右至少各有四个中性字符；A、B 字符不得出现在中性上下文中。

### 平衡方式

| 检查项 | 结果 |
|---|---:|
| 形近字对 | 2,162 |
| A/B 页面组 | 2,162 |
| 图片 | 4,324 |
| 匹配目标位置 | 17,296 |
| 训练事件 | 34,592 |
| 字体 | 4 |
| 图像条件 | 清晰、轻旧化 |
| A / B 作为正确字的出现次数 | 17,296 / 17,296 |

每个字对在每个字体和图像条件组合中恰好出现一次。事件进入辅助损失后再次按原始事件均衡，避免一个字符因为 byte 分支较多而获得更大梯度预算。

### 第二阶段合并

| 组成 | Train | Dev |
|---|---:|---:|
| 第一阶段完整 replay | 18,800 | 960 |
| 新增形近字 A/B 页面 | 4,324 | 0 |
| 第二阶段合计 | 23,124 | 960 |

开发集保持第一阶段冻结的 960 条，不把形近字训练 A/B 页面或真实评估集写入开发集。

## 标签与训练元数据

所有 GT 都由生成时使用的文本直接产生：

- 通用样本的标签来自训练侧字符与版式规则。
- 图像状态变化继承原始文本，不能改写标签。
- A/B 页面分别使用各自实际渲染的目标字符作为标签。
- 形近字事件额外记录正确字、rival、目标位置和 byte-token 对齐信息。

训练前检查 tokenizer 无损往返、未知 token、图片路径、空标签、目标位置和 A/B 配对关系。第一阶段正式训练包共检查 19,760 条 train+dev 记录，tokenizer 往返失败和未知 token 均为 0。

## 质量与隔离检查

| 检查 | 要求 |
|---|---|
| 缺图 / 空 GT | 必须为 0 |
| 重复 ID | 必须为 0 |
| 图片与标签映射错误 | 必须为 0 |
| A/B 非目标区域变化 | 必须为 0 |
| A/B 方向曝光差 | 必须为 0 |
| 训练与封存合成验证图片哈希重合 | 必须为 0 |
| 评估来源标识命中 | 必须为 0 |
| 评估图片、GT、预测或错误进入构建 | 禁止 |

第二阶段正式 A/B 数据审计结果中，非目标邻域变化为 0，封存合成验证集图片哈希重合为 0，评估来源标识命中为 0。

## 发布边界

- Hugging Face 模型仓库只发布完整合并权重、必要配置、tokenizer、哈希清单和模型卡。
- GitHub 保存训练策略、数据构建说明和必要的复现入口，不把训练图片放入模型仓库。
- 真实评估集单独发布和维护，不与训练数据混放。
- 合成数据只能用于训练覆盖，不能代替真实图片上的最终评估。
