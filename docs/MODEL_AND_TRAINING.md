# 模型与训练策略

本文档说明 `NuosuBburma OCR` 的模型来源、两阶段训练方法、评估口径和结论边界。

## 模型信息

| 项目 | 内容 |
|---|---|
| 中文名 | 规范彝文 OCR |
| 基座模型 | `PaddlePaddle/PaddleOCR-VL-1.6`（0.9B） |
| 基座版本 | `66317acc4c9fc17bd154591ce650735cd2855f3e` |
| 微调方式 | 两阶段 rank-16 LoRA |
| 最终发布形式 | 完整合并模型，推理时不需要额外 LoRA |
| 识别范围 | 规范彝文、汉字、拉丁字母、数字和标点的端到端混合文字 OCR |

训练数据和评估数据严格分离：

| 数据 | 用途 | 是否进入训练 |
|---|---|---|
| 第一阶段合成训练集 | 建立通用混合文字与图像状态适应能力 | 是 |
| 第二阶段合成形近字集 | 补充成对形近字细粒度视觉监督 | 是 |
| 合成开发集 | 训练前后独立检查与流程验收 | 否 |
| 1,030 张真实评估图片及 GT | 最终性能评估 | 否 |

两个训练阶段均未使用评估图片、评估 GT、评估预测、评估错误或评估统计构建、筛选或调整训练样本。

## 两阶段训练

| 阶段 | 主要目标 | 训练起点 |
|---|---|---|
| 第一阶段：通用混合文字适配 | 学习规范彝文及彝汉、拉丁、数字、标点混排，并覆盖字体、版式和图像退化 | 固定版本的 PaddleOCR-VL-1.6 |
| 第二阶段：形近字专项继续微调 | 在保持整页 OCR 能力的同时，补充彝文形近字 A/B 对照监督 | 第一阶段完成 588/588 steps 后的正确合并导出模型 |

第二阶段不是从裸基座重新训练，也不是从其他形近字实验分支继续训练。最终发布权重由“第一阶段合并模型 + 第二阶段 LoRA”正确合并得到。

## 第一阶段：通用混合文字适配

第一阶段使用独立构建的纯合成数据。文字来源限定为规范彝文 Unicode 字表、常用汉字表和训练侧规则；图像由锁定字体、版式和退化规则生成，不读取真实评估材料。

### 数据组成

| 组成 | Train | Dev | 作用 |
|---|---:|---:|---|
| 通用核心 | 18,000 | 800 | 彝文、汉字、混排、字体、版式、形近字覆盖和图像状态 |
| Latin 补充 | 400 | 80 | 大小写、姓名、年份、缩写以及彝/汉混排中的拉丁文字 |
| 残墨与真字退化成对样本 | 400 | 80 | 区分外部墨点、残墨和真实字形，降低伪字符插入风险 |
| 合计 | 18,800 | 960 | 第一阶段正式训练与开发规模 |

带圈编号 `①-⑳` 注入通用核心，不额外增加样本数。通用核心使用四款彝文字体，并覆盖单行、区域和页面式版面。彝文形近字不是第二阶段才加入：第一阶段已把全部 2,162 个形近字对按 3 款字体和 2 个上下文变体注入普通 OCR 样本，共形成 12,972 个覆盖事件，并使用完整 OCR 序列交叉熵训练。

### 图像状态

- 保留清晰、轻旧化和重但可读的训练条件。
- 禁用容易产生不可控形变的 shear 和 gamma。
- 重退化样本占比固定为 30%；轻/重退化最多使用 1/2 个核心操作。
- 对人工确认过重的七类退化路径只降低强度，不删除类别：一类固定为 0.25，其余六类固定为 0.80。
- 成对残墨样本只改变图像状态，GT 保持逐字节一致。

### 训练参数

| 项目 | 设置 |
|---|---:|
| LoRA rank | 16 |
| 每卡 batch size | 4 |
| 梯度累积 | 16 |
| 有效 batch size | 64 |
| Epoch | 2 |
| Optimizer steps | 588 |
| 学习率 / 最低学习率 | `5e-4` / `5e-5` |
| Scheduler / warmup | cosine / 0.01 |
| 精度 | bf16 |
| 主损失 | 完整 OCR 自回归交叉熵 |

第一阶段完成后先导出并合并 LoRA，所得模型作为第二阶段唯一训练起点。

## 第二阶段：形近字专项继续微调

第一阶段之后，模型的错误没有发散，而是集中在少数彝文形近字关系——典型表现是“倾向于多加一笔或一点”（如 ꀉ ↔ ꀊ）。第二阶段不是从裸基座重训，而是从第一阶段完成 588/588 步后的合并导出模型出发，新建 rank-16 LoRA，以更低学习率和新的训练参数继续微调。与此同时，第二阶段训练集对第一阶段的 18,800 条训练样本做旧数据回放（rehearsal / replay），并新增 4,324 张独立合成的形近字 A/B 页面，总规模为 23,124；开发集仍为 960。继承模型权重与回放旧训练数据是同时采用的两项措施，并不冲突。

### A/B 对照数据

形近字集合由规范彝文字形独立构建，不根据评估错误选字，包括多字体字形相似度聚类 1,657 对、声调弧语言规律 342 对、单字体稳定关系 122 对和 coverage-floor 覆盖补足 41 对，共 2,162 对（构建细节见 [训练数据构建报告](TRAINING_DATA_CONSTRUCTION_REPORT.md)）。正式构建另有 2,162 组 A/B 匹配页面，每组两张图、每张图 8 行，每行安排一个形近字对；形近字对按固定规则跨页面排程，并非一个字对对应一组页面：

- A/B 两张图的每组对应行保持背景、上下文、位置、字体和图像条件一致，只替换该行的目标字符。
- 每张图的 8 行对应四款彝文字体和清晰/轻旧化两种条件，同一页内不重复使用形近字对。
- 每个字对在每个字体与图像条件组合中恰好出现一次。
- A 作为正确字和 B 作为正确字的曝光次数严格相等。
- 目标字左右至少各保留四个中性上下文字符，形近字本身不进入中性上下文。

最终构建包含 2,162 组页面对、4,324 张图片、17,296 个匹配位置和 34,592 个形近字训练事件。

### 训练目标

**分词分析。** 锁定的 PaddleOCR-VL-1.6 tokenizer 把全部 1,165 个规范彝文字符固定拆成 3 个 byte token。在 2,162 个形近字对中，没有字对在第 1 个 token 分叉；1,637 对在第 2 个 token 分叉，525 对到第 3 个 token 才分叉。也就是说，区分形近字的信息集中在字符 byte 序列后段的少数 token 上。这一观察用于组织第二阶段的局部监督；形近字集合本身仍由字形与彝文字符体系构建，不由 token 相似度发现。

基于此，第二阶段按数据类型使用两种交叉熵范围，并在声明的目标字符位置加入轻量辅助约束：

~~~text
总损失
= CE
+ 0.001 × event-balanced byte-path loss
+ 0.0005 × full-byte pairwise proxy
+ 0.1 × non-target consistency
~~~

- 再次加入的第一阶段样本计算完整 OCR 序列 CE；新增 A/B 页面只对每页 8 个目标字符的 24 个 byte token 计算 target-only CE。
- `byte-path loss` 在正确字符实际经过的 byte 分支上，与形近字表中尚未被前序 token 排除的一跳邻居组织局部竞争。
- `full-byte pairwise proxy` 比较正确字符与声明 rival 所有实际不同的 byte 位置，使用正确字符的 teacher-forced 前缀逐 byte 评分；它不是推理阶段的完整字符联合概率排序。
- `event-balanced` 先在同一原始事件内部平均，再在事件之间平均，避免多分支字符获得额外权重。
- `non-target consistency` 限制 A/B 配对对非目标位置的扰动，不使用第一阶段模型作为教师。
- 第一阶段训练样本的完整序列 CE 用于维持混合文字、标点、换行和生成长度能力。

上述辅助项没有完成逐项开关消融，因此最终改善不能单独归因于某一个辅助损失。

第二阶段也没有完成“只用 4,324 条 A/B”与“回放 18,800 条旧样本并加入 A/B”的独立对照，因此不能宣称旧数据回放的占比已经最优。

### 训练参数

| 项目 | 设置 |
|---|---:|
| Train / Dev | 23,124 / 960 |
| 新增形近字图片 | 4,324 |
| LoRA rank | 16 |
| 每卡 batch size / 梯度累积 | 4 / 16 |
| 有效 batch size | 64 |
| Epoch | 1 |
| Optimizer steps | 362 |
| 学习率 / 最低学习率 | `1e-4` / `1e-5` |
| Scheduler / warmup | cosine / 0.03 |
| 更新范围 | 主要限制在视觉侧与视觉到文本投影路径，语言层冻结 |

## 评估结果

最终合并模型在完整 1,030 张真实扫描或实际拍摄图片上统一评估。公开结果沿用答辩材料的逐样本平均 NED；每张图片等权，不按字符长度加权。模型、标注和预测文件均以 SHA-256 锁定。

![最终模型总体、场景与难度 NED](figures/evaluation-performance-overview.svg)

总彝文 NED 为 `0.0432`，去空白平均 NED 为 `0.0399`，原始平均 NED 为 `0.0474`。按场景和难度拆分后，书籍扫描与低难度样本最稳定，手写拍照与高难度样本误差最高。规范化后完全一致为 333 / 1,030（`32.3%`）；语料级 CER `6.22%` 只作为补充，不替代逐样本平均 NED。完整公式、场景、难度和异常分析见 [最终模型评估分析](EVALUATION_RESULT_ANALYSIS.md)。

在训练阶段，48 页文档真实诊断子集用于比较训练：第一阶段到最终模型的平均 NED 从 `0.0069` 降至 `0.0056`（下降 18.4%），彝文 NED 从 `0.0241` 降至 `0.0216`（下降 10.2%）；总编辑错误由 117 降至 95，字符替换由 114 降至 89，但插入与删除由 3 增至 6，锁定形近字替换仅由 34 降至 31。按页比较为 18 页改善、23 页持平、7 页退化。

因此，第二阶段可以归因到总体替换减少，但不能宣称提升了生成长度稳定性，也不能在没有开关消融的情况下宣称形近字辅助项具有独立贡献。

## 使用边界

- 支持单行、文本区域和整页图片；书籍扫描与清晰印刷最稳定。
- 复杂整页、屏幕照片、实景照片和长手写段落误差更高。
- 个别高频单向形近字混淆仍然存在。
- 当前 0.9B 模型是高精度参考模型，尚未针对移动端和端侧运行优化。
