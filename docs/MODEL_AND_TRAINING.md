# 模型与训练策略

本文档说明 `NuosuBburma OCR` 的模型来源、两阶段训练方法、评估口径和结论边界。

## 模型信息

| 项目 | 内容 |
|---|---|
| 中文名 | 规范彝文 OCR |
| 基座模型 | `PaddlePaddle/PaddleOCR-VL-1.6`（0.9B） |
| 基座版本 | `66317acc4c9fc17bd154591ce650735cd2855f3e` |
| 微调方式 | 两阶段 rank-16 LoRA |
| 最终发布形式 | 完整合并模型，推理时不需要额外 LoRA |
| 识别范围 | 规范彝文、汉字、拉丁字母、数字和标点的端到端混合文字 OCR |

训练数据和评估数据严格分离：

| 数据 | 用途 | 是否进入训练 |
|---|---|---|
| 第一阶段合成训练集 | 建立通用混合文字与图像状态适应能力 | 是 |
| 第二阶段合成形近字集 | 补充成对形近字细粒度视觉监督 | 是 |
| 合成开发集 | 训练前后独立检查与流程验收 | 否 |
| 1,030 张真实评估图片及 GT | 最终性能评估 | 否 |

两个训练阶段均未使用评估图片、评估 GT、评估预测、评估错误或评估统计构建、筛选或调整训练样本。

## 两阶段训练

| 阶段 | 主要目标 | 训练起点 |
|---|---|---|
| 第一阶段：通用混合文字适配 | 学习规范彝文及彝汉、拉丁、数字、标点混排，并覆盖字体、版式和图像退化 | 固定版本的 PaddleOCR-VL-1.6 |
| 第二阶段：形近字专项继续微调 | 在保持整页 OCR 能力的同时，补充彝文形近字 A/B 对照监督 | 第一阶段完成 588/588 steps 后的正确合并导出模型 |

第二阶段不是从裸基座重新训练，也不是从其他形近字实验分支继续训练。最终发布权重由“第一阶段合并模型 + 第二阶段 LoRA”正确合并得到。

## 第一阶段：通用混合文字适配

第一阶段使用独立构建的纯合成数据。文字来源限定为规范彝文 Unicode 字表、常用汉字表和训练侧规则；图像由锁定字体、版式和退化规则生成，不读取真实评估材料。

### 数据组成

| 组成 | Train | Dev | 作用 |
|---|---:|---:|---|
| 通用核心 | 18,000 | 800 | 彝文、汉字、混排、字体、版式、形近字覆盖和图像状态 |
| Latin 补充 | 400 | 80 | 大小写、姓名、年份、缩写以及彝/汉混排中的拉丁文字 |
| 残墨与真字退化成对样本 | 400 | 80 | 区分外部墨点、残墨和真实字形，降低伪字符插入风险 |
| 合计 | 18,800 | 960 | 第一阶段正式训练与开发规模 |

带圈编号 `①-⑳` 注入通用核心，不额外增加样本数。通用核心使用四款彝文字体，并覆盖单行、区域和页面式版面。彝文形近字在该阶段作为通用字符覆盖的一部分进入训练。

### 图像状态

- 保留清晰、轻旧化和重但可读的训练条件。
- 禁用容易产生不可控形变的 shear 和 gamma。
- 重退化样本占比固定为 30%；轻/重退化最多使用 1/2 个核心操作。
- 对人工确认过重的七类退化路径只降低强度，不删除类别：一类固定为 0.25，其余六类固定为 0.80。
- 成对残墨样本只改变图像状态，GT 保持逐字节一致。

### 训练参数

| 项目 | 设置 |
|---|---:|
| LoRA rank | 16 |
| 每卡 batch size | 4 |
| 梯度累积 | 16 |
| 有效 batch size | 64 |
| Epoch | 2 |
| Optimizer steps | 588 |
| 学习率 / 最低学习率 | `5e-4` / `5e-5` |
| Scheduler / warmup | cosine / 0.01 |
| 精度 | bf16 |
| 主损失 | 完整 OCR 自回归交叉熵 |

第一阶段完成后先导出并合并 LoRA，所得模型作为第二阶段唯一训练起点。

## 第二阶段：形近字专项继续微调

第一阶段之后，模型的错误没有发散，而是集中在少数彝文形近字关系——典型表现是“倾向于多加一笔或一点”（如 ꀉ ↔ ꀊ）。第二阶段针对这类细粒度视觉差异继续微调：在第一阶段的 18,800 条训练数据上完整 replay，并加入 4,324 张独立合成的形近字 A/B 页面，总训练规模为 23,124；开发集仍为 960。

### A/B 对照数据

形近字集合由规范彝文字形在多字体条件下独立构建，不根据评估错误选字，来自两条线索：多字体字形相似度聚类，以及彝文语言规律（如“次高调”与对应“中平调”字形相同、仅差一笔）。共覆盖 2,162 个形近字对，每对生成 A、B 两张匹配页面（构建细节见 [训练数据构建报告](TRAINING_DATA_CONSTRUCTION_REPORT.md)）：

- A/B 两张图保持背景、上下文、位置、字体和退化条件一致，只替换目标字符邻域。
- 使用四款彝文字体和清晰/轻旧化两种条件。
- 每个字对在每个字体与图像条件组合中恰好出现一次。
- A 作为正确字和 B 作为正确字的曝光次数严格相等。
- 目标字左右至少各保留四个中性上下文字符，形近字本身不进入中性上下文。

最终构建包含 2,162 组页面对、4,324 张图片、17,296 个匹配位置和 34,592 个形近字训练事件。

### 训练目标

规范彝文字符通常经 byte-fallback tokenizer 拆成多个 token。第二阶段在完整 OCR 交叉熵之外，仅在声明的目标字符位置加入轻量辅助约束：

~~~text
总损失
= 完整 OCR CE
+ 0.001 × event-balanced byte-path loss
+ 0.0005 × 完整字符竞争约束
+ 0.1 × non-target consistency
~~~

- `byte-path loss` 在正确字符实际经过的 byte 分支上组织候选竞争。
- 完整字符竞争约束比较正确字符与对应形近候选的完整字符得分。
- `event-balanced` 先在同一原始事件内部平均，再在事件之间平均，避免多分支字符获得额外权重。
- `non-target consistency` 限制 A/B 配对对非目标位置的扰动。
- 全文 OCR CE 和第一阶段 replay 保留，用于维持混合文字、标点、换行和生成长度能力。

上述辅助项没有完成逐项开关消融，因此最终改善不能单独归因于某一个辅助损失。

### 训练参数

| 项目 | 设置 |
|---|---:|
| Train / Dev | 23,124 / 960 |
| 新增形近字图片 | 4,324 |
| LoRA rank | 16 |
| 每卡 batch size / 梯度累积 | 4 / 16 |
| 有效 batch size | 64 |
| Epoch | 1 |
| Optimizer steps | 362 |
| 学习率 / 最低学习率 | `1e-4` / `1e-5` |
| Scheduler / warmup | cosine / 0.03 |
| 更新范围 | 主要限制在视觉侧与视觉到文本投影路径，语言层冻结 |

## 评估结果

最终合并模型在 1,030 张真实扫描或实际拍摄图片上的结果如下，NED 越低越好：

| 指标 | 结果 |
|---|---:|
| Raw Avg NED | `0.047445` |
| 去空白 Avg NED（诊断） | `0.039896` |
| NFKC + 去空白 Avg NED（诊断） | `0.034604` |

Raw Avg NED 使用原始解码字符串，不删除空白，也不做 Unicode 兼容规范化。后两项只用于分离内容错误、空白差异和 Unicode 表示差异，不替代 Raw Avg NED。

在 48 页人工 GT 诊断子集（《凉山彝语语法》整页）上，可直接比较第一阶段模型与最终模型。逐页平均 NED：

| 训练阶段 | 平均 NED | 彝文 NED |
|---|---:|---:|
| 第一阶段 | 0.0069 | 0.0241 |
| 第二阶段（最终） | 0.0056 | 0.0216 |

平均 NED 下降 18.4%，彝文 NED 下降 10.2%。错误结构对比：

| 训练阶段 | 总编辑错误 | 字符替换 | 插入+删除 | 锁定形近字替换 |
|---|---:|---:|---:|---:|
| 第一阶段 | 117 | 114 | 3 | 34 |
| 第二阶段 | 95 | 89 | 6 | 31 |

按页比较，最终模型相对第一阶段有 18 页改善、23 页持平、7 页退化。第二阶段降低了总编辑错误和字符替换，收益主要来自替换减少；但插入与删除由 3 增至 6，因此不宣称第二阶段提升了生成长度稳定性。锁定形近字替换仅由 34 降至 31，改善幅度明显小于总替换下降，说明困难彝文形近字尚未被系统解决。形近字辅助项应视为小权重正则，在未完成开关消融前不宣称其独立贡献。

## 使用边界

- 整幅图片端到端识别，无需切行或切块；书籍扫描与清晰印刷最稳定。
- 复杂整页、屏幕照片、实景照片和长手写段落仍需人工复核。
- 个别高频单向形近字混淆仍然存在。
- 当前 0.9B 模型是高精度参考模型，尚未针对移动端和端侧运行优化。
