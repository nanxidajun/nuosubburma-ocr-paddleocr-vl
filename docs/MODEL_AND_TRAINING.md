# 模型与训练说明

本文档说明提交模型的基础设置、三阶段 LoRA 微调策略、训练包构成、分支选择和当前评估结果。英文缩写只在必要处保留，正文优先使用中文说明。

先给出边界：训练包、开发诊断集和最终评估集分开使用。

| 材料 | 用途 | 是否进入训练 |
|---|---|---|
| 训练包 | 给模型提供学习信号，包含真实训练材料、训练侧合成样本、文本不变的视觉变化样本和少量约束样本 | 是 |
| 开发诊断集 | 比较不同训练分支的输出表现，观察编辑距离、公式化片段、多余拉丁字母和异常长输出等风险 | 否 |
| 最终评估集 | 报告提交模型效果，支撑公开结果表；最新口径为 `758` 条真实样本 | 否 |

训练包约束：

- 真实训练材料和合成样本分开记录。
- 视觉变化样本只改变字体、清晰度和旧印刷状态，不改变文本标签。
- 拉丁注音、脚注、多行区域等高风险样本设置上限。
- 评估集图片、样本编号和标准答案不进入训练包。
- 新样本来自训练侧字符规则、频率规则、版式规则或原始合成标签。

训练数据的详细来源、标签来源和质检见 [训练数据构建报告](TRAINING_DATA_CONSTRUCTION_REPORT.md)。

## 三阶段调优策略

三阶段调优要体现的是训练策略，不是三次堆数据。

| 阶段 | 解决的问题 | 数据动作 | 判断方式 |
|---|---|---|---|
| 第一阶段：单书可行性 | 验证 `PaddleOCR-VL-1.6` + LoRA 能否学习规范彝文 | 使用《勒俄特依》真实裁切行建立基本任务 | 看模型是否能学会规范彝文字形和基本输出格式 |
| 第二阶段：覆盖补充 | 解决低频字符、字体变化、旧印刷退化和彝汉混排覆盖不足 | 加入训练侧合成样本、字符覆盖样本和视觉变化样本 | 分开观察彝文、汉字、数字、混排和异常输出 |
| 第三阶段：分支检查 | 比较不同训练包分支是否真正更稳 | 使用固定开发诊断集读取模型输出，不改写训练包 | 同时看编辑距离、公式化片段、多余拉丁字母、异常长输出和区域漏行 |

这条路线的关键是分工清楚：

- 第一阶段证明路线可行。
- 第二阶段补训练覆盖，但控制高风险通道比例。
- 第三阶段只做分支比较，不把开发诊断集或最终评估集写回训练包。

## 分支选择

分支比较主要改变数据分布，不改变模型结构。

![Training branch strategy](figures/training_branch.svg)

| 分支 | 训练数据动作 | 开发诊断集结果 | 判断 |
|---|---|---|---|
| 早期稳定分支 | 稳定数据形状，补多行区域与纯彝文稳定样本 | 整体 `0.0634`，彝文 `0.0512`，汉字 `0.0265`，异常长输出 `0` | 早期稳定基线 |
| 版式与注音再平衡分支 | 恢复字典注音、单行彝汉混排和纯彝文字符覆盖 | 整体 `0.0505`，彝文 `0.0473`，汉字 `0.0261`，异常长输出 `0` | 总分继续下降，但脚注风险仍高 |
| 当前提交分支 | 文本不变、图像状态变化样本 `9069` 行，拉丁注音不放大，脚注/多行区域设上限 | 整体 `0.0342`，彝文 `0.0372`，汉字 `0.0225`，公式化片段 `2`，异常长输出 `0` | 当前提交模型 |
| 长尾格式对照分支 | 追加 `350` 行格式长尾约束样本 | 整体 `0.0429`，公式化片段回到 `10`，异常长输出 `1` | 风险回升，保留为对照 |

结论：继续加数据不一定变好。提交模型在开发诊断集上同时降低整体、彝文、汉字三组编辑距离，并保持较低输出风险。

## 当前评估结果

最终 `758` 条整合评估集作为未微调基座与 LoRA 微调模型的统一评估口径。未微调基座和 LoRA 微调后结果均已完成；LoRA 结果已合并最新人工 GT 修正。归一化编辑距离（NED）越低，说明输出越接近人工标注。

| 指标 | 未微调基座 | LoRA 微调后 |
|---|---:|---:|
| 评估样本 | 758 | 758 |
| 平均归一化编辑距离（Avg NED，越低越好） | 0.726733 | 0.070342 |
| 忽略空白后的平均编辑距离 | 0.719600 | 0.069978 |
| 规范化并忽略空白后的平均编辑距离 | 0.706794 | 0.069796 |
| Exact | 0 / 758 | 447 / 758 (59.0%) |
| 只看彝文字符的平均编辑距离 | 1.000000 | 0.069870 |
| 只看汉字的平均编辑距离 | 0.209245 | 0.055882 |
| 只看数字的平均编辑距离 | 0.369451 | 0.260416 |
| 替换符 / 公式化片段 / 多余拉丁字母 / 异常长输出 | 16 / 105 / 321 / 34 | 0 / 6 / 1 / 1 |

## 使用边界

- 本模型可以输入整页、区域和行图。
- 当前最稳定的使用方式通常是单行图和区域图识别。
- 复杂整页直接识别时，容易在长段落和复杂混排处产生非标注换行、阅读顺序偏差或段落边界误判。
- 复杂整页、手写拍照长段落、多栏或脚注/注音密集页面，建议先走页面切割，再做文本区域识别、页面文本合并和人工核验。
- 手写拍照样本已纳入独立观察，结果与印刷体分开解读。
- 脚注符号有 `2` 条公式化片段输出需要单独核验。
- 本版本尚未进行专门的端侧或移动端优化。
