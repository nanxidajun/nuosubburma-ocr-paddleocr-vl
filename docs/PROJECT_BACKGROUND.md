# 项目背景与任务定义

本文档对应主提交稿中的“项目概览、场景稀缺性与任务复杂度”。它回答三个问题：为什么规范彝文属于本次比赛值得覆盖的长尾 OCR 场景、这个任务难在哪里、模型最终要解决什么实际问题。

## 项目概览

`规范彝文 OCR / NuosuBburma OCR` 基于 `PaddleOCR-VL-1.6 (0.9B)` 进行 LoRA 微调，是一个面向规范彝文（ꆈꌠꁱꂷ / Nuosu Bburma）真实场景的 PaddleOCR-VL 衍生模型。

项目目标是把书籍扫描页、裁切行图、文字区域、规范手写稿和彝汉混排材料中的规范彝文转成可复制、可检索、可继续加工的 Unicode 文本。它既服务于文献数字化，也服务于学习、检字、注音、校对和后续 NLP 语料建设。

这个项目从真实旧书出发。先让模型学会规范彝文字形，再补页面切割、识别、混排、输出稳定性和评估复查这些实际环节。它关注的是一批真实资料能否被持续整理、复查和复用，而不只是某一张样例图能不能跑通。

## 场景稀缺性

规范彝文诞生于 1980 年，已经形成稳定的标准字符体系，承载了教材、诗歌、民族研究文献和日常教学资料。但在数字化层面，大量资料仍停留在图片、扫描件和纸质书稿中，无法稳定搜索、复制和整理。

这类场景的稀缺性不只来自“数据少”，还来自工具链断裂：通用 OCR 通常不认识规范彝文，公开评估集又很少，研究者和学习者即使有资料，也常常只能人工录入。OCR 在这里不是锦上添花，而是资料进入数字世界的第一道门。这正是 PaddleOCR 衍生模型挑战赛适合覆盖的长尾场景：真实需求明确，通用模型覆盖不足，开源基线价值高。

这会带来几个直接问题：

| 问题 | 影响 |
|---|---|
| 学习依赖检字 | 学习者需要反复查字形、读音和含义，学习成本高 |
| 资料难以复制 | 教材、旧书和文献不能方便整理、引用和再加工 |
| 检索能力缺失 | 民族文献和教学资料难以进入数字检索系统 |
| 后续 NLP 缺语料 | 转写、注音、翻译和语言模型训练缺少高质量文本入口 |
| 通用 OCR 基本不可用 | 现有通用 OCR 很难直接识别规范彝文，往往需要重新建立数据和评估入口 |

这也是项目最初的起点：可用工具很少，但真实资料数字化、教学检字和语料建设的需求一直在。一个可复现的规范彝文 OCR 基线，能把零散的图片资料变成后续校对、注音、检索和训练都能继续接手的文本入口。

## 任务定义

```text
输入：包含规范彝文或规范彝文混排内容的图像
输出：图像中可见文本的 Unicode 转写，尽量保留混排关系和基本标点
```

真实使用路径如下：

```text
用户上传 PDF 或图片
-> 页面渲染或读取
-> 自动页面切割或人工裁剪文字区域
-> 对 line / region 执行 OCR，必要时单独观察 page OCR
-> 导出文本、CSV 或 JSON
-> 需要时继续做注音和人工校对
```

本模型可以输入整页图像、文本区域和裁切行图。当前最稳定的使用方式通常是 `line / region OCR`；直接 `page OCR` 用于观察整页边界、阅读顺序和换行风险，不作为复杂整页稳定交付的主要路径。复杂整页文档在版面较密、规范手写、多栏、脚注、注音块或图文混排较强时，建议配合页面切割流程或人工复核。

这样的定义看起来比“端到端整页 OCR”保守，但更贴近真实工作流：先把最可用的 line/region 能力做稳，再用页面切割和结构标注把整页问题逐步拆开。对低资源文字来说，能被复查、能被接力，比一次性包装成完整产品更重要。

## 任务复杂度

规范彝文 OCR 的复杂度来自真实资料本身。单行识别准确，不等于真实资料整理可用。

| 复杂度 | 具体表现 | 对模型的影响 |
|---|---|---|
| 字符复杂度 | 规范彝文覆盖 1165 个字符，形近字多，差异常集中在细小笔画、方向、开口和局部结构 | 低分辨率、旧印刷和压缩会放大形近字混淆 |
| 视觉域复杂度 | 新印刷 PDF、旧书扫描、页面照片、手机照片、规范手写材料噪声不同 | 模型不能只适应单一字体或单一本书 |
| 结构复杂度 | 真实资料包含整页、区域、单行、短诗式排布、脚注块、注音块和段落图 | 多行区域容易漏行，复杂整页需要页面切割配合 |
| 混排复杂度 | 彝文、汉语、数字、脚注符号、拉丁注音和标点会同时出现 | 容易出现 LaTeX 化、ASCII 尾巴、长输出等输出空间漂移 |

评估和选模时，这些问题都会单独看。清晰单行只是其中一种输入，不能代表真实资料整理。项目保留 line、region、page、手写、旧印刷和混排分组，就是为了让每一种能力各自暴露，而不是被一个平均分掩盖。

## 当前能力边界

- 可以输入 page / region / line 图像。
- 清晰印刷行图和常见彝汉混排是当前最稳定场景。
- 复杂整页直接识别容易出现非标注换行、阅读顺序偏差或段落边界误判，建议配合页面切割流程或人工复核。
- 规范手写资料已纳入独立观察，结果与印刷体分开解读。
- 尚未进行专门的端侧/移动端优化。
