# 评估集说明

最终评估使用 [NuosuBburma OCR Evaluation Set](https://huggingface.co/datasets/nanxidajun/NuosuBburma-OCR-Evaluation-Set)，以该 Hugging Face 数据集仓库的当前版本为准。全部样本来自真实扫描或实际拍摄，**不含合成评估数据**，也不作为训练或开发数据。

## 评估任务

输入为单张包含规范彝文、彝汉混排及少量拉丁字母、数字或标点的图像。模型按视觉阅读顺序转写可见文字，保留必要的行结构和原有字符，不翻译、不改写，也不补全图中不存在的文字。评估重点是复杂版式、混排文字和真实拍摄干扰条件下的识别能力。

## 内容统计

| 项目 | 数量 |
|---|---:|
| 样本 / 图片 | 1,030 / 1,030 |
| 文档页 / OCR 实例 | 519 / 511 |

统计口径：“文档页”对应 `sample_type=page`，“OCR 实例”对应 `sample_type=region`；图像获取方式由 `scene` 独立记录。

## 难度分类依据

样本按视觉与版式难度分为三级（低 271 / 中 501 / 高 258）：

- **低难度**：结构较简单的文档区域样本，裁切范围集中，背景和版式干扰较少。
- **中难度**：普通文档整页及结构较复杂的区域样本，包含多行文本或常规复杂版式，需要处理阅读顺序、段落结构及页面噪声。
- **高难度**：标题封面及全部拍照样本（手写、屏幕、实景），可能包含复杂背景、透视畸变、摩尔纹、反光、阴影、褶皱、污渍及图文混排等干扰。

## 数据文件与标注格式

| 文件 | 用途 |
|---|---|
| `annotations.jsonl` | 正式样本标注，每行一个图像与对应 GT |
| `images/` | 与标注一一对应的评估图片 |
| `dataset_summary.json` | 数据分布、文件完整性与哈希审计摘要 |

每行一个 JSON 对象，包含 `id`、`images`、`messages` 和 `meta`；GT 位于唯一一条 `assistant` 消息中，只记录图中可见文字。

## 标注原则

标准答案（GT）只记录图中**可见文字**，按视觉阅读顺序转写规范彝文、汉字、拉丁字母、数字和标点，保留必要的行结构和原有字符，**不翻译、不改写、不补全**图中不存在的内容。图像退化或噪声不改变 GT。每条样本只有一份正式 assistant GT。

## 分级人工质检流程

正式 GT 采用分级人工质检体系，模型输出只作为预标注和差异提示，不能自动覆盖人工结论：

1. **预标注**——先生成预标注，但不把预标注直接作为最终标准答案。
2. **多版本交叉检查**——用多轮训练得到的不同模型版本产生识别结果，辅助发现潜在标注错误。
3. **字符级对齐高亮**——通过字符级对齐工具高亮替换、漏字和多字位置，集中检查模型输出与当前标注的差异。
4. **首轮人工核查**——项目负责人对照原始图片检查高亮出的差异并修正错误。
5. **母语者复核**——将疑难、不确定或容易混淆的规范彝文样本交由熟悉规范彝文的母语者核查。
6. **人工裁决为准**——最终标准答案由人工确认，以原图和人工判断为准。

[在线可视化](https://huggingface.co/spaces/nanxidajun/NuosuBburma-OCR-Evaluation-Set) 提供图片与标准答案的逐条对照，便于公开复查。

## 训练隔离

- 评估图片、GT、预测、错误类型和统计**不进入**训练或开发数据。
- 训练生成器不读取评估目录、书名、样本 ID 或图像裁切。
- 形近字集合、字体、上下文、退化条件和训练权重不依据评估错例选择。
- 评估集只在模型训练和导出完成后用于报告结果。

## 报告边界

本文档只陈述已执行的质检流程与已通过的发布审计。母语者复核针对疑难、不确定或易混的规范彝文样本，非全量；不含尚未统计的标注一致率、纠错条数或复核轮数等指标，也不声称双盲独立标注。

## 使用边界

- 评估集只用于独立评估和错误分析，不作为训练或开发数据。
- 拍摄类图像由数据集制作者实际采集；书籍页面的原作品权利归相应权利人所有。
- 图片和标注的公开使用须遵守相关权利要求及适用法律。
- 完整图片、标注与统计以 Hugging Face 数据集仓库的当前版本为准；分组结果分析见 [评估结果分析](EVALUATION_RESULT_ANALYSIS.md)。
