## 构成与难度

| 维度 | 分布 |
|---|---|
| 样本类型 | `line 515`，`region 84`，`page 4` |
| 场景 | 新印刷扫描 `309`，旧书扫描 `235`，规范手写 `53`，实拍/屏幕 `6` |
| 文字混合 | `yi 321`，`yi_han 265`，`yi_han_latin 17` |
| 数字 | 含数字 `81`，不含数字 `522` |
| 难度 | `easy 172`，`medium 324`，`hard 107` |

难度分层不按模型输出倒推，而按来源、视觉质量、输入粒度、版面结构和文字混合情况确定：

- `easy`：清晰行图、单行、版面边界简单、干扰少。
- `medium`：彝汉混排、旧印刷、区域样本、数字/标点/注音邻近、普通真实照片。
- `hard`：整页、旧书页面、长段落、目录/脚注/页码、复杂实拍标牌和规范手写 region/page。

复杂整页压力测试材料与主评估集分开维护，不放入本评估集目录，也不混入 `603` 条主结果。

## 标注与质检

评估集采用模型辅助闭环，但模型输出只作为人工校对草稿：

```text
真实来源图片整理
-> 页面切分、区域裁剪或整页保留
-> 中间模型生成预标注草稿
-> 人工核对并固化标准答案
-> 记录来源、场景、粒度、难度和文字混合
-> 生成 annotations.jsonl、samples.csv 和 dataset_summary.json
```

关键规则：

- 未经人工确认的模型输出不作为 gold label。
- 合成样本不进入正式评估集。
- 评估集样本不作为最终训练目标。
- 多行 region 保留换行关系，复杂整页不与清晰单行混为同一种能力。
- 彝文、汉字、数字、Latin 注音和标点分别观察，不把所有错误混成一个指标。

## 下载

复跑评估时，可以下载到本地 `datasets/` 目录：

```bash
# 国内网络较慢时，取消下一行注释使用 Hugging Face 镜像：
# export HF_ENDPOINT=https://hf-mirror.com

hf download nanxidajun/NuosuBburma-OCR-Evaluation-Set \
  --repo-type dataset \
  --local-dir datasets/NuosuBburma_OCR_Evaluation_Set
```

`datasets/` 是本地下载目录，已在 `.gitignore` 中忽略，不作为 GitHub 仓库内容提交。

## 任务定义

输入：包含规范彝文或规范彝文混排内容的图像。

输出：图片中可见文本的 Unicode 转写，尽量保留混排关系和基本标点。

推荐提示词：

```text
<image>OCR:
```

## 数据边界

- 本数据集用于评估，不是训练集。
- 样本来自真实材料，不使用合成样本作为主评分材料。
- 书籍样本来源于扫描件裁剪，原始出版物版权归原出版社和权利人所有。
- 使用时请同时尊重原始材料版权与本项目的数据说明。
